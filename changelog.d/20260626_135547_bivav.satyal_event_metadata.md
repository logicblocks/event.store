### Added

- Added an optional `metadata` field to `NewEvent` and `StoredEvent`, backed by a `metadata JSONB NOT NULL` column in the `events` table. The field accepts an arbitrary JSON-serialisable mapping and defaults to `None` (no metadata).

### Changed

- **Breaking change — migration required for existing deployments.** Adding the `metadata JSONB NOT NULL` column requires the following migration:

  ```sql
  ALTER TABLE events ADD COLUMN metadata JSONB NOT NULL DEFAULT '{}';
  ALTER TABLE events ALTER COLUMN metadata DROP DEFAULT;
  ```

  The transient `DEFAULT '{}'` backfills all existing rows during the `ADD COLUMN` step. The default is then dropped so the column's final state matches the canonical no-default DDL, ensuring all new rows must supply an explicit value.
